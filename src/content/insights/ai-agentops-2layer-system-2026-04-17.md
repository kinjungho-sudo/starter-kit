# 비개발자 AI 에이전트 1개월 실전 운용기 - Claude Code + OpenClo 2층 시스템

> 비개발자 AI 교육 강사/유튜버가 1개월간 업무의 80%를 AI 에이전트로 전환한 실전 경험. 어떤 도구가 아니라 어떻게 설계하는가에 집중한 구체적 사례.

---

## 핵심 인사이트

**에이전트 시대, 업무의 핵심이 "실행"에서 "설계"로 바뀐다.**
무엇이 가능한지만 알면 AI가 데려다 준다. 비개발자도 전문 영역 진입 가능.

---

## 2층 에이전트 시스템 구조

```
맥미니 (24시간 상시 가동) - 텔레그램으로 원격 지시
├── 업무 비서: Claude Code (18개 프로젝트)
│   └── 정밀한 하네스 엔지니어링 → 안정적, 복잡한 업무 처리
└── 일상 비서: OpenClo
    └── 느슨한 하네스 + Computer Use → 유연, 즉흥적 지시
```

**Claude Code vs OpenClo 선택 기준:**
| | Claude Code | OpenClo |
|---|---|---|
| 하네스 | 빡빡하게 조일 수 있음 | 느슨하게 유연 |
| 적합 용도 | 반복 업무, 정밀 처리, 품질 담보 | 즉흥 지시, Computer Use, 빠른 시도 |
| 비유 | 일잘러 팀원 | 뭐든 시킬 수 있는 비서 |

## Claude Code 에이전트 18개 구성

### 자비스 (라우터 에이전트)
- 역할: 전체 에이전트 분배 허브
- 텔레그램 창구 → 적절한 프로젝트 라우팅

### 교육 설계 그룹
- 고객사명 + 직무 수준 입력 → 맞춤형 AI 실습 시나리오 10개 자동 생성
- 수강생 결과물 평가 + 개인별 피드백 + 결과 보고서 생성
- **이전 작업 시간**: 몇 시간 → **지금**: 자동

### 강의 슬라이드 그룹
- HTML 생성 → PPT 변환 → 구글 드라이브 업로드 (약 10분)
- 스타일 가이드 기반, 일정 자율성 부여
- **오토리서치 적용**: 1회→10회 반복 개선 후 품질 극적 향상

### 리서치 & 모니터링
- 링크드인 15인 계정 모니터링 → 매일 어제 글 브리핑
- 경쟁 유튜브 채널 모니터링 → 영상 주제/썸네일 아이디어 추천
- 카톡 나에게 보내기 링크 → 자동 정리 사이트 (ReadWise 벤치마킹)

### 영업 & 행정
- 이메일 분류 → 노션 DB 자동 업데이트 → 매일 아침 영업 건 리마인드
- 세금계산서 자동 발행 ("○○에 200만원 세금계산서 끊어줘")

### 개발 도구
- 크롤링 에이전트: 쿠팡 리뷰, 나라장터 공고 수집 (옥토파스보다 뛰어남, 무료)
- 유튜브 편집: 무음/불필요한 말 제거 러프컷

## OpenClo 일상 비서 활용

### 캘린더 관리
```
텔레그램: "다음주 화요일 오후 2시 김민수씨 티타임 한시간 잡아줘"
→ 구글 캘린더 자동 등록
→ 매일 9시 오늘 일정 브리핑
```

### 카카오톡 나에게 보내기 아카이빙
- 하루 2회 카톡 나에게 보내기 내역 자동 수집
- 링크 방문 → 내용 보강 → 구글 시트 정리
- 바이브 코딩 웹사이트에서 학습 콘텐츠로 재소비

### KTX 좌석 감시 (창의적 구현)
- 코레일 웹사이트: 자동화 철저히 차단
- **우회 방법**: 안 쓰는 안드로이드폰 + ADB + scrcpy → 모바일 앱 제어
- 텔레그램: "오송→서울 내일 오전 9~10시 조회" → 빈 좌석 모니터링

> 카카오톡 API 없어도, 코레일이 막아도 - **사람처럼 동작하는 에이전트는 막을 수 없다**

## 핵심 방법론 4가지

### 1. 하네스 엔지니어링
- **정의**: CLAUDE.md + 스킬 + 서브에이전트 + 코드 스크립트 + 규정 문서로 모델을 빡빡하게 규율
- **비유**: 적토마(뛰어난 AI) + 안장(하네스) = 내 의도대로 달리는 말
- GPTs는 프롬프트만으로 규율 → 에이전트 하네스는 코드+문서로 더 정밀

### 2. 오토리서치 (Auto Research)
- **출처**: 안드레 카파시 방법론
- **방법**: AI에게 평가 기준 제시 → 스스로 실행-평가-개선 루프 반복
- **적용 결과**: PPT 에이전트 1회→10회 자동 개선 후 품질 극적 향상
- **시사점**: 직장인 사무 노동의 상당수는 평가 가능한 지표로 AI가 스스로 개선 가능

### 3. 깃허브 벤치마킹
- 깃허브 = 클로드 코드가 가장 잘 따라하는 형태의 업무 안내서
- **방법 1**: 원하는 기능 깃허브 링크 → "이것처럼 만들어줘"
- **방법 2**: 심층 리서치로 관련 깃허브 레포 먼저 찾기
- **방법 3**: 유튜브 자막/논문/블로그 텍스트 → "이 접근법 적용해줘"

### 4. Claude Code + Codex 교차 비판
- 두 코딩 에이전트가 서로 다른 의견 제시
- Codex가 전체 구조 개선에 더 탁월
- Codex 답변을 Claude Code에 붙여넣어 토론 유도 → 내적 완결성 향상

## 1개월 운용 후 솔직 평가

**좋은 점:**
- 업무 80% AI 에이전트 지원 달성
- 비개발자가 CAD 자동화, 수량산출서, 건설/제조 분야 컨설팅까지 진입
- "무엇이 가능한지만 알면 AI가 데려다 준다" - 전문 분야 장벽 붕괴

**어려운 점:**
- 에이전트 많아지면 읽을 자료, 판단할 것도 많아짐 → 피로도 증가
- 유지보수 필요: API 변경, 웹사이트 구조 변경으로 파이프라인 고장
- 에이전트 관리 자체가 새로운 업무 (팀장/대표의 관리 업무와 유사)
