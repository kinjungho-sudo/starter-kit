# AI 디자이너의 새로운 표준 - DESIGN.md

> 유훈식 교수, 브런치 (2026-04-09)
> **픽셀을 그리는 시대에서 의도를 선언하는 시대로.**

---

## DESIGN.md란?

> "인공지능 에이전트가 이해할 수 있는 언어로 작성된 평문 마크다운 형식의 디자인 시스템 문서"

| 항목 | 기존 방식 | DESIGN.md |
|------|----------|-----------|
| 형식 | 피그마 라이브러리, JSON 스키마 | 마크다운 (.md) |
| AI 이해 | ❌ 파싱 어려움 | ✅ LLM이 직접 처리 |
| 버전 관리 | 별도 툴 필요 | Git으로 관리 |
| 이식성 | 툴 종속 | 모든 텍스트 에디터 |

---

## 왜 마크다운인가?

1. **LLM 친화성** - 구조화된 텍스트를 AI가 가장 잘 처리
2. **무의존성** - 빌드 도구 없이 어디서든 열람 가능
3. **Git 버전 관리** - 디자인 변경 이력 추적 가능

---

## DESIGN.md의 3가지 핵심 이점

### 1. 제약 조건을 통한 품질 향상
- AI가 명시적으로 정의된 색상·간격 규칙을 준수
- → 브랜드 일관성 자동 유지

### 2. 살아있는 문서화
- 전문 개발 지식 없이도 수정 가능
- → 수정 즉시 AI 워크플로우에 반영

### 3. 도구 간 연결성
- 피그마 → Claude Code → 커서로 파편화 없이 디자인 전이
- → "디자인 하네스" 역할

---

## DESIGN.md 9가지 구성 요소

```
1. 시각적 테마 및 분위기   ← 감성/톤 정의
2. 색상 팔레트 및 역할     ← Primary/Secondary/Neutral
3. 타이포그래피 규칙       ← 폰트 패밀리, 크기 스케일
4. 컴포넌트 스타일링       ← 버튼, 카드, 인풋 등
5. 레이아웃 및 간격        ← Grid, Spacing 토큰
6. 깊이 및 고도            ← Shadow, z-index 체계
7. 권장 및 금지 사항       ← Do/Don't 명시
8. 반응형 동작             ← Breakpoint별 규칙
9. 에이전트 프롬프트 가이드 ← AI에게 줄 힌트
```
