# AI 에이전트 학습 데이터 완전 가이드 - 수집·전처리·관리

## 핵심 요약
AI 에이전트의 품질은 프롬프트가 아니라 **학습 데이터의 품질**이 결정한다.
데이터는 구조화/비구조화 두 유형이 있고, 수집 방법은 실제환경·피드백·합성 3가지다.
좋은 데이터 → 전처리 → 지속 관리의 사이클이 에이전트 성능을 복리로 높인다.

## 주요 내용

### 1. 학습 데이터의 두 가지 유형

| 유형 | 특징 | 활용 |
|------|------|------|
| **구조화 데이터** | DB 정리된 숫자·날짜·문자열 | 규칙 기반 솔루션, 정확한 분석 |
| **비구조화 데이터** | 텍스트·이미지·비디오·SNS | 자연어 처리, 이미지 인식 능력 |

- 두 유형을 **균형 있게** 학습시켜야 다양한 도메인에 적용 가능한 유연성 확보
- 구조화의 정확성 + 비구조화의 풍부함이 결합될 때 복잡한 작업 수행 가능

### 2. 데이터 수집 3가지 방법

- **실제 환경 데이터**: 고객 서비스 대화 기록, 센서 데이터 등 현장 데이터 → 실용성 최고
- **인간 피드백 (RLHF)**: 에이전트 행동에 인간이 평가 → 정렬(Alignment) 핵심 수단
- **합성 데이터**: 실제 데이터가 부족할 때 인공 생성 → 희귀 케이스 커버 가능

### 3. 전처리의 핵심 단계
- 오류·중복 제거 → 데이터 일관성 확보
- **비식별화**: 개인정보 제거 (필수 법적 요건)
- 필요 정보 추출 → 학습에 불필요한 노이즈 제거
- 전처리 품질이 이후 모든 학습 과정의 신뢰성을 결정

### 4. 학습 데이터 관리 전략 3축
- **지속적 업데이트**: 최신 정보 반영 → 모델이 현실과 괴리되지 않게
- **인간 피드백 통합**: 사용자 반응으로 지속 품질 개선
- **도메인 특화 데이터 구축**: 일반 데이터보다 도메인 특화 데이터가 성능 훨씬 높음

### 5. 실제 활용 사례
- **금융**: 거래 패턴 학습 → 이상 거래·사기 탐지 정확도 향상
- **의료**: 방대한 의료 데이터 패턴 인식 → 진단 보조

## 인사이트

### 핵심 교훈
- "AI 에이전트가 얼마나 똑똑한가"는 **데이터 파이프라인의 품질**에 달려 있다
- 프롬프트 엔지니어링보다 데이터 엔지니어링이 더 근본적인 레버
- 합성 데이터 활용 → 실제 데이터 부족 문제를 해결할 수 있는 실용적 방법

### 코마인드웍스 사업 연결
- **LLM Wiki 시스템 자체가 학습 데이터 파이프라인**: 수집 → 전처리(품질검사) → 관리(유통기한/점검) 구조가 이 가이드의 실천
- 클라이언트 에이전트 구축 시: "어떤 데이터를 어떻게 수집할지" 설계 단계에서 이 프레임 적용
- **콘텐츠 소재**: "AI 에이전트가 멍청한 진짜 이유 - 프롬프트가 아니라 데이터 문제" → 유튜브 훅으로 활용 가능
